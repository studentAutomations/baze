Understanding CONSTRAINT in SQL

Definition

A constraint in SQL is a rule that restricts the type of data that can be stored in a table. It ensures that the data adheres to certain standards and maintains the integrity of the database. In Oracle SQL, constraints can be applied at both the column level and the table level.

Types of Constraints

  • NOT NULL: Ensures that a column cannot contain NULL values.
  • UNIQUE: Guarantees that all values in a column are distinct.
  • PRIMARY KEY: A combination of NOT NULL and UNIQUE; it uniquely identifies each row in a table and prevents duplicate entries.
  • FOREIGN KEY: Establishes a relationship between two tables, ensuring referential integrity.
  • CHECK: Validates that values in a column meet a specific condition.

Application in the Code

CREATE TABLE RADI_NA
(
 MBR INTEGER NOT NULL,
 BRPR INTEGER NOT NULL,
 SATI NUMBER(2,0),
 CONSTRAINT RADI_NA_PK PRIMARY KEY (MBR, BRPR)
);

In your code snippet:

  • The CONSTRAINT RADI_NA_PK PRIMARY KEY (MBR, BRPR) line defines a primary key constraint named RADI_NA_PK. This means:
    • The combination of MBR and BRPR must be unique for each row in the RADI_NA table.
    • Neither MBR nor BRPR can contain NULL values, enforcing that every record must have valid identifiers.

Importance of Constraints

Constraints are crucial for maintaining data accuracy and integrity. They prevent invalid data entries and ensure that relationships between different pieces of data are consistent. For instance, without the primary key constraint, duplicate or NULL values could lead to ambiguity when retrieving records from the database.

In summary, constraints like those defined in your SQL code are essential tools for ensuring that your database remains reliable and that the data it contains is valid and consistent.
